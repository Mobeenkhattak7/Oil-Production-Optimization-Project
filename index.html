Use Case: Oil Production Optimization Using Sensor & Historical Data

1. Use Case Selection

Modern oil wells generate terabytes of unstructured time-series data from sensors measuring pressure, temperature, flow rates, and gas lift information. Optimizing production requires analyzing this complex data to detect inefficiencies, predict underperformance, and recommend corrective actions. This project aligns with the 3W multivariate sensor dataset containing real, simulated, and expert-labeled time-series data.

2. Business Problem Statement

The goal is to develop a data-driven pipeline to analyze oil well sensor data and identify operational inefficiencies early, so engineers can optimize production and reduce downtime.

3. Dataset Information

  • Dataset: 3W Dataset (Oil Well Multivariate Time-Series)
  • Source: Petrobras offshore wells
  • Time-series sequences: ~1,984 sequences with 8 sensor variables + label
  • Labels: 0 = Normal, 1–8 = Undesirable operating events

4. Big Data Pipeline

The pipeline includes real-time ingestion (Kafka), batch processing (PySpark, Airflow), storage (HDFS/S3), and visualization (Power BI/Tableau/Grafana).

5. Machine Learning Methodology

We use LSTM/GRU for sequential modeling of well dynamics, combined with XGBoost for classification of undesirable events. Feature engineering includes rolling statistics, FFT components, and physical well features.

6. Implementation Plan

  • Tools: PySpark, TensorFlow/PyTorch, XGBoost, MLflow, Kafka, Hadoop/S3
  • Batch pipeline: Data cleaning → feature extraction → LSTM embeddings → XGBoost classification
  • Real-time pipeline: Kafka ingestion → preprocessing → anomaly scoring → classification → alerts
  • Evaluation: Precision, Recall, F1-score, False Alarm Rate, Latency

Conclusion

This project integrates big data engineering with ML-based time-series modeling to optimize oil production. Using the 3W dataset and a scalable analytics architecture, the solution detects undesirable events early and supports operational decisions.
