Oil Production Optimization Using Sensor & Historical Data

Big Data Analytics Project — Powered by the 3W Dataset

Overview Dataset Pipeline Machine Learning Implementation

1. Project Overview

This project aims to optimize oil well production using multivariate sensor data collected from offshore oil wells. Using big data engineering and machine learning, the system identifies inefficiencies, predicts undesirable events, and enables proactive operational decision‑making.

2. Dataset Description

  • Name: 3W Dataset (Petrobras Offshore Wells)
  • Instances: ~1,984 multivariate sequences
  • Features: 8 sensor readings (pressure, temperature, GL flow, choke data)
  • Labels: 0 = normal, 1–8 = undesirable events
  • Types: real, simulated (OLGA), and expert‑drawn event patterns

3. Big Data Architecture

  • Kafka-based real-time stream ingestion
  • Raw + Processed storage layers (HDFS / S3)
  • Spark-based batch preprocessing and feature extraction
  • Cassandra/Redis feature store for real-time lookups
  • Visualization dashboards (Grafana, Power BI, Tableau)

4. Machine Learning Workflow

  • Time-series cleaning and interpolation
  • Windowed sequence creation
  • Feature engineering: statistical, FFT, physical indicators
  • Model: LSTM/GRU + XGBoost hybrid classifier
  • Evaluation metrics: F1-score, recall, early detection rate

5. Implementation Plan

  • Batch training pipeline (Spark + MLflow)
  • Real-time inference pipeline (Kafka + streaming model)
  • Deployment using Docker + cloud hosting
  • Continuous monitoring: drift, accuracy, anomaly frequency
